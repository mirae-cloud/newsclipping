# 뉴스클리핑 봇 — 제작 지시서 (Claude Code용)

> 이 문서 하나로 프로젝트 전체를 이해하고 만들 수 있도록 작성됨.
> 아래 스펙을 그대로 따라 구현하고, 애매한 부분은 이 문서의 "라우팅 규칙"과 "기술 주의사항"을 우선 참고할 것.

---

## 0. 프로젝트 개요

**무엇을**: 매일 오전 7시(KST)에 국내·글로벌 뉴스와 경제지표를 자동으로 수집·요약·분석하여, ① 웹사이트로 보여주고 ② 이메일 한 통으로 발송하는 개인용 뉴스클리핑 봇.

**누구를 위해**: 대기업·컨설팅·외국계 취업을 준비하는 사용자. 산업군별 동향, Business(딜·전략 등) 영역, 거시 경제 흐름을 매일 빠르게 파악하는 것이 목적.

**핵심 원칙**:
- 웹사이트는 뉴스/지표 API를 **직접 호출하지 않는다.** 매일 백그라운드 "엔진"이 데이터를 만들어 파일로 저장하고, 웹사이트는 그 파일만 읽어 렌더링한다. (API 키 노출 방지 + 무료 티어 제약 회피)
- 각 뉴스는 **제목(원문 링크) + 요약 bullet 2~3개 + 인사이트(파급효과 분석)** 구조.

## 1. 전체 아키텍처

```
[스케줄러: 매일 오전 7시 KST]
        │ (실행)
        ▼
[수집·요약 엔진]  ── 뉴스/지표 API 호출 → 24시간 필터 → 중복 제거 → Gemini로 요약·분류·인사이트 생성
        │ (저장)
        ▼
[데이터 파일: JSON]  (economy.json / domestic.json / global.json)
        │ (읽기)
        ├────────────► [웹사이트: 3개 탭]
        └────────────► [이메일: Gmail SMTP로 1통 통합 발송]
```

- **스케줄러**: GitHub Actions cron (무료). 정확한 시각이 아니라 수 분~수십 분 지연될 수 있음(데일리 클리핑엔 무방).
- **엔진**: Python 권장 (뉴스 API·경제 API·Gemini API 호출, 데이터 가공, 이메일 발송을 한 스크립트로).
- **웹사이트**: 정적 사이트(HTML/CSS/JS 한 벌). GitHub Pages / Vercel / Netlify 중 하나로 무료 호스팅.

## 2. 데이터 소스 및 API

### 2-1. 국내 뉴스 — 네이버 검색 API(메인) + 구글 뉴스 RSS(보충)

- **네이버 검색 API (메인)** — 무료. 네이버 개발자센터에서 Client ID/Secret 발급.
  - 문서: https://developers.naver.com/docs/serviceapi/search/news/news.md
  - 반환: 제목, **원문 링크(originallink)**, 요약(description), 발행일시(pubDate). 최신순(sort=date) 지원.
  - 원문 링크를 바로 주기 때문에 구글 RSS처럼 리다이렉트 해제가 필요 없어 훨씬 빠름 — 이 속도 차이 때문에 메인 소스로 채택(2026-07 수정).
- **구글 뉴스 RSS (보충)** — API 키 불필요, 무료. 네이버 결과가 부족할 때만 보충 호출.
  - 키워드 검색 피드: `https://news.google.com/rss/search?q={검색어}&hl=ko&gl=KR&ceid=KR:ko`
  - 검색어 자리에 한국어 키워드를 그대로 URL 인코딩해서 넣음. 검색 피드는 쿼리당 최대 약 100건 반환.
  - **주의(중요)**:
    - RSS 링크는 원문이 아니라 **구글 경유 리다이렉트 링크**로 나옴 → 이메일/사이트에 원문 URL을 넣으려면 리다이렉트를 한 번 풀어주는 처리 필요(단순 HTTP 리다이렉트가 아니라 구글 내부 RPC를 거쳐야 해서 기사당 수십 초가 걸릴 수 있음 — **최종 선택된 기사에 한해서만** 해제 처리할 것).
    - **비공식 엔드포인트**(구글이 공식 문서화·지원하는 기능 아님). 대량·고빈도 호출 시 일시 차단 가능성이 있음(공식 한도 미명시, **추정**).
- 두 소스 결과는 **제목·URL 기준으로 중복 제거** 후 통합.

### 2-2. 글로벌 뉴스 — Currents API

- 무료 티어가 넉넉한 편(가입 직전 최신 한도 재확인 권장). 신용카드 불필요·상업적 사용 허용으로 알려짐.
- 키워드·언어·날짜 필터 지원. 영어 위주.

### 2-3. 경제지표

**표시할 지표는 3종 — (1) 기준금리, (2) 물가, (3) 환율 — 을 한국·미국 기준으로만 다룬다.** (주가지수·국채 등은 지표 카드에서 제외. 단 코스피·S&P500 등 시장 뉴스는 아래 '경제 뉴스클리핑'의 키워드로 다뤄짐.)

- **국내 — 한국은행 ECOS OpenAPI** (무료). 회원가입 시 인증키 자동 부여(보통 1일 내 활성화).
  - 문서/키 신청: https://ecos.bok.or.kr/api/
  - 사용 지표: **한국 기준금리, 한국 소비자물가지수(CPI), 원/달러 환율.**
  - 시계열은 통계검색(기간 지정) 엔드포인트로 조회 → 5년치 히스토리·평균 계산 가능.
- **미국 — FRED API** (무료, 120 req/분). 계정 로그인 후 인증키 발급.
  - 문서/키: https://fred.stlouisfed.org/docs/api/fred/
  - 사용 시리즈 코드: **`FEDFUNDS`(미국 기준금리), `CPIAUCSL`(미국 CPI).**
  - 환율은 원/달러 하나로 한·미를 함께 보므로 ECOS의 원/달러 환율로 충분(미국 측 별도 환율 지표 불필요).

### 2-4. 요약·분류·인사이트 — Google Gemini API (무료 티어)

- 뉴스 요약(제목 1줄 + bullet), 카테고리 분류, 인사이트 생성, 전체/영역별 헤드라인 추출에 사용.
- Google AI Studio 무료 티어 사용 → 봇 운영비 $0. 신용카드 불필요.
- SDK: 신형 `google-genai` (`pip install google-genai`). 구형 `google-generativeai`는 폐기 → 사용 금지.
- 모델: 기본값 **`gemini-3.5-flash`**. 2026-04-01부로 Google AI Studio 무료 티어는 Flash·Flash-Lite 계열만 제공.
  **주의(중요, 확인 필요)**: 2026-07 실측 결과 이 프로젝트/키는 `gemini-3.5-flash`와 `gemini-2.5-flash` **둘 다 동일하게 하루 20건 한도**에 걸림 —
  모델 선택의 문제가 아니라 **결제 계정 미연결 등 프로젝트 차원의 제한**일 가능성이 높다. 공개 문서상 Flash 계열은 하루 1,500건이 일반적이라고
  알려져 있어 이 20건은 비정상적으로 낮음. 실제 사용 전 Google AI Studio에서 프로젝트 결제 상태·한도를 확인할 것.
  모델명은 상수/환경변수로 분리해 언제든 교체 가능하게 구현.
- 키: 환경변수 `GEMINI_API_KEY`로 주입(하드코딩 금지). 429(rate limit) 대비 **자체 분당 호출 수 제한(스로틀)** + 서버가 알려주는 retryDelay 우선 재시도 적용.
- 문서/키: https://aistudio.google.com , https://ai.google.dev/gemini-api/docs/quickstart
- 주의(데이터): 무료 티어는 프롬프트가 구글 제품 개선(학습)에 쓰일 수 있음(한국 사용자 해당). 공개 뉴스라 민감도는 낮음.
- 주의(한도): 무료 한도·모델명은 자주 바뀌므로 구현 직전 https://ai.google.dev/gemini-api/docs/pricing 에서 재확인.

## 3. 준비물 체크리스트 (사용자가 발급/설정할 것)

코딩이 아니라 "가입 후 클릭" 작업들. 발급한 키는 모두 **GitHub Secrets**(또는 로컬 `.env`)에 넣고, 코드에 직접 쓰지 말 것.

- [ ] 네이버 개발자센터: Client ID / Secret
- [ ] Currents API 키
- [ ] 한국은행 ECOS 인증키
- [ ] FRED API 키
- [ ] Google AI Studio (Gemini) API 키 — https://aistudio.google.com 에서 발급. 카드/과금 불필요. 새 키는 'Gemini API 전용'으로 제한(restrict) 권장.
- [ ] Gmail 앱 비밀번호 (Gmail에서 2단계 인증 활성화 후 발급 — SMTP 발송용)
- [ ] GitHub 계정 + 저장소
- [ ] 호스팅 계정 (Vercel/Netlify) 또는 GitHub Pages 활성화

## 4. 경제 탭 상세 (① 지표 + ② 경제 뉴스)

경제 탭은 두 부분으로 구성한다: **(1) 지표 카드**(API 숫자) + **(2) 경제 뉴스클리핑**(키워드 기반, 산업군/Business와 동일 형식).

### 4-1. 지표 카드 (API)
각 지표 카드는 **위쪽에 숫자 3개, 아래쪽에 5년 꺾은선 그래프**.

- 숫자 3개: **오늘(최신값) / 1달 단순평균 / 1년 단순평균**
  - '평균'은 전부 **단순평균(달력 기준)** 으로 계산. (동향만 보는 용도이므로 거래일 가중 불필요)
  - 발표 주기 차이 반영: 환율은 일 단위 → 최근 30일/365일 단순평균. CPI·기준금리는 월/비정기 → 최신 발표값 및 최근 12개월 평균.
- 5년 라인차트: 웹사이트에서 차트 라이브러리(예: Chart.js)로 렌더.
- **표시 지표(총 5개 카드)**:
  - 기준금리 — 한국(ECOS) / 미국(FRED `FEDFUNDS`)
  - 물가(CPI) — 한국(ECOS) / 미국(FRED `CPIAUCSL`)
  - 환율 — 원/달러(ECOS) 1개

### 4-2. 경제 뉴스클리핑
- **8번의 '경제' 키워드 그룹**을 버튼으로 삼아, 산업군/Business 영역과 **동일한 방식**(제목+bullet 2~3개+인사이트)으로 주요 경제 뉴스를 정리한다.
- 소스는 국내·글로벌 뉴스에서 '경제' 키워드로 필터링(예: "코스피 3,000 돌파", "Fed 금리 동결" 등).
- 상단 summary 박스도 국내/글로벌 탭과 동일하게 둔다(그날 경제 핵심 bullet + 그룹별 헤드라인).

## 5. 뉴스 항목 구조 & 인사이트 프롬프트

### 5-1. 각 뉴스 항목의 출력 구조
1. **카테고리당 기사 개수** - 한 카테고리당 기사는 3개 이하. 규모, 변화 정도, 산업 내 위치 등을 고려하여 가장 핵심적이고 중요한 기사 순서대로 선별. 다른 기사에 비해 상대적으로 중요도가 낮으나, 추가 확인할 경우 도움이 되는 뉴스 주제는 3개 기사 출력 후 최하단에 주제당 15자-20자 이내 수준으로 별도 기재.
2. **제목** — 뉴스 핵심 한 줄 요약. **원문 링크를 제목에 삽입.**
3. **요약 bullet 2~3개** — 주요 내용 요약.
4. **인사이트** — 아래 지시에 따른 파급효과 분석.

### 5-2. 인사이트 생성 시 Gemini에게 줄 지시 (프롬프트에 포함)
- 해당 산업 **내부** 영향뿐 아니라, **후방산업(원재료·소재 공급단)** 과 **전방산업(최종 수요단)** 으로의 파급을 짚을 것.
- **경기민감 산업(부동산·건설 등)** 은 거시 흐름과 연결해 해석할 것.
- **타 산업으로의 대체/보완 효과**를 최소 1개 제시할 것.
- **일견 무관해 보이는 산업과의 연결**이 있다면 제시할 것 (예: 여행·항공 침체 ↔ 홈엔터·게임 수요 증가).
- **확실한 인과가 아니면 반드시 '추정' 또는 '가능성'으로 표시**할 것. (인사이트는 추론이므로 과잉 단정 금지 — 특히 '무관해 보이는 연결'은 사변적이라 이 표시가 필수)

## 6. 산업군 (16개) — 키워드·라우팅

> 아래 키워드는 **시작점(seed)**. 운영하며 노이즈가 많은 키워드는 빼거나 좁혀 튜닝할 것.

1. **금융** — 은행, 금융지주, 대출, 카드, 보험, 증권, 금융규제
2. **TMT (기술 & 미디어 & 통신)** — 통신, 미디어, 플랫폼, 콘텐츠, 클라우드, 소프트웨어, 빅테크
3. **로봇 & 반도체** — 반도체, 파운드리, HBM, 메모리, 로봇, 로보틱스, 자동화
4. **헬스케어 & 라이프사이언스** — 제약, 바이오, 신약, 의료기기, 병원, 임상, 헬스케어
5. **소비재 & 리테일 (B2C)** — 소비재, 유통, 리테일, 이커머스, 식품, F&B, 화장품, **농업·농산물(전량 여기)**
6. **여행 & 항공** — 여행, 관광, 여객 항공, 호텔, 항공권
7. **자동차 & 모빌리티** — 자동차, 전기차/EV, 완성차, 모빌리티, 자율주행
8. **산업재 (Industrial Goods)** — 철강, 비철금속, 기계, 장비, 부품, 조선, 중공업 *(물리·금속 밸류체인, 아래 재정의 참조)*
9. **방산** — 방산, 국방, 무기, 항공우주, 방위산업
10. **운송 & 물류** — 물류, 화물, 해운, 택배, 공급망 운송 *(B2B 유통 포함)*
11. **건설 & 인프라** — 건설, 건설사, 수주, 인프라, 플랜트, SOC
12. **부동산** — 부동산, 아파트, 분양, 리츠, 상업용부동산, 주택시장
13. **소재 & 화학** — 석유화학, 나프타, 기초화학, 배터리소재, 양극재, 리튬, 희토류, 신소재 *(화학 밸류체인, 아래 재정의 참조)*
14. **석탄 & 석유 & 가스** — 석유, 원유, 천연가스, LNG, 정유, 유가 *(연료·자원 자체)*
15. **에너지 & 전력** — 발전, 전력, 원자력, 태양광, 풍력, 신재생, 송전, 한전 *(발전~송·배전 인프라)*
16. **공공 & 사회 & 교육** — 정부정책, 공공, 규제, 교육, 사회복지, 비영리

### 6-1. 산업군 라우팅 규칙 (경계 기사 분류)
- **소재 & 화학(13) vs 산업재(8)** — "화학 vs 물리"로 구분:
  - 13 = 원자재 → **1차 가공(예: 나프타)** → 석유화학·기초화학 → 첨단화학소재·신소재. "화학적으로 만들어지는 소재."
  - 8 = 금속 원자재·중간재(철강 등) + 부품·기계·장비. "물리적으로 가공·조립되는 재화."
- **광물 채굴(Metals & Mining) 분기** — 산출물 성격 기준:
  - 철광석 등 **금속 광물** → **산업재(8)** (철강 계열로 이어짐)
  - 리튬·희토류 등 **배터리·첨단소재용 광물** → **소재 & 화학(13)**
- **석탄 & 석유 & 가스(14) vs 에너지 & 전력(15)** — "연료·자원 자체(14)" vs "그 연료로 전기를 만드는 발전·전력망(15)". 예: LNG 수입·가격 → 14, LNG 발전소 가동 → 15.
- **농업** — 별도 카테고리 없음. **전량 소비재 & 리테일(5)** 로.

## 9. 웹사이트 구조 (UI/UX 상세)

### 9-0. 디자인 톤
- **매거진형**: 화이트 배경 + 넉넉한 여백, 절제된 타이포그래피 중심(Substack·뉴스레터 느낌). 텍스트(요약·인사이트) 가독성이 최우선.
- **강조색**: 1~2개의 포인트 컬러만 사용하고 나머지는 무채색 톤. 강조색 적용 대상 —
  - 활성 탭/버튼 상태
  - 인사이트 문단의 좌측 보더 또는 배경(요약 bullet과 시각적으로 구분되도록)
  - 저장(북마크) 아이콘의 활성 상태
  - (2026-07 수정) 인사이트 안의 "추정"/"가능성" 단어 자체를 별도 뱃지로 강조하지는 않음 — 문구는 유지하되 시각적 강조는 생략.
- 구체적 색상 코드는 구현 단계에서 시안으로 확정.

### 9-1. 최상단 탭 4개
`[경제]` `[산업군]` `[Business]` `[저장됨]`

- **경제 탭**: 4번 스펙 그대로 = (1) 지표 카드 5개(기준금리·물가 한/미 + 원/달러 환율, 각 숫자 3개 + 5년 라인) + (2) 경제 뉴스클리핑(경제 키워드 그룹 버튼 + summary 박스). 경제 뉴스는 국내·글로벌 소스를 **통합**하여 보여줌(별도 토글 없음).
- **산업군 탭**: 진입 시 상단 좌측 정렬로 **전체 Summary**(그날 산업 전반 핵심 bullet + 16개 카테고리 중 주요 헤드라인)와 **16개 카테고리 버튼**을 표시. 카테고리 버튼 클릭 시:
  - Summary는 유지된 채, 바로 아래 **[국내 | 글로벌] 토글**이 나타남 (해당 산업군 뉴스는 국내·글로벌 소스가 섞여 있으므로 토글로 필터링).
  - 그 아래 해당 카테고리의 **Sub-summary**(카테고리 핵심 bullet) + **기사 목록**(제목+bullet+인사이트) 표시.
- **Business 탭**: 산업군 탭과 동일한 구조, 11개 카테고리 버튼.
- **저장됨 탭**: 사용자가 북마크한 기사 전문 목록. 각 뉴스 카드에 북마크 아이콘을 두어 저장/해제 토글. 날짜가 지나 원본 JSON이 갱신·삭제되어도 저장된 기사는 localStorage에 전문(제목·링크·bullet·인사이트·날짜·카테고리)이 보존되어 계속 조회 가능.

### 9-2. Toggle 정리 (총 2종)
1. **국내/글로벌 필터 토글** — 산업군·Business 카테고리 진입 후, 하위 카테고리 버튼 클릭 시 노출.
2. **뉴스 카드 펼치기/접기 토글** — 기본은 제목만 표시, 클릭 시 bullet 2~3개 + 인사이트가 아코디언으로 펼쳐짐. 한 화면에 더 많은 기사를 배치하기 위함.

### 9-3. 데이터/저장소 원칙
- 뉴스·지표 **원본 데이터**는 브라우저 저장소(localStorage 등)에 캐싱 금지 — 항상 매일 갱신되는 JSON 파일에서 읽음.
- 예외적으로 **저장(북마크) 기능**은 localStorage에 기사 전문을 저장하는 것을 허용(9-1 참고). 이는 데이터 소스 캐싱이 아니라 사용자가 명시적으로 선택한 개인 아카이브이므로 원칙에 저촉되지 않음.
- 과거 날짜 클리핑의 자동 아카이브·조회 기능은 만들지 않음(오늘자만 표시). 다만 엔진 산출 JSON 자체는 날짜별로 별도 보관해도 무방(추후 확장 여지).

## 10. 이메일 구조 (Gmail SMTP, 1통 통합)

- **발송 방식**: 엔진 실행 마지막 단계에서 Gmail SMTP로 발송. Gmail 2단계 인증 활성화 후 **앱 비밀번호**로 로그인(구글이 '보안 수준 낮은 앱' 방식을 2025년 5월 폐지). 개인 Gmail은 하루 500통 한도 → 하루 1통엔 무관.
- **구성 순서(한 통 통합)**: ① **경제**(맨 앞: 지표 요약 = 기준금리·물가 & 환율 한/미 + 경제 뉴스클리핑 주요 건) → ② **국내 뉴스** → ③ **글로벌 뉴스**.
- 각 뉴스는 5-1 구조(제목 링크 + bullet + 인사이트) 그대로.
- **HTML 이메일은 심플한 인라인 스타일**로 작성(이메일 클라이언트 호환성).

## 11. 스케줄링 & 배포

- **스케줄러**: GitHub Actions cron으로 **매일 오전 7시(KST)** 에 엔진 실행 → 데이터 파일 생성/갱신 → (필요 시) 사이트 재배포 + 이메일 발송.
  - GitHub Actions cron은 UTC 기준. KST(UTC+9) 오전 7시 = 전날 22:00 UTC → **`0 22 * * *`** 로 설정.
  - 주의: cron은 정확한 분이 아니라 수 분~수십 분 지연될 수 있음(7시 정각이 아니라 7시 언저리 실행). 데일리 클리핑엔 무방하나, "정확히 7시"가 꼭 필요하면 이 방식으로는 보장되지 않음(**참고**).
- **호스팅**: GitHub Pages / Vercel / Netlify 중 하나(무료).
- **비용**: 전부 무료 티어로 운영 → 사실상 $0. (뉴스·경제 API 무료 + Gemini LLM 무료 + GitHub Actions 무료 + 정적 호스팅 무료.) 단, 각 무료 티어의 한도·조건은 변동될 수 있으므로 주기적으로 확인.

## 12. 기술 주의사항 (Gotchas)

- 웹사이트는 정적 — 뉴스/지표 API를 브라우저에서 직접 호출하지 말 것(키 노출·CORS 문제). 반드시 엔진이 만든 JSON만 읽을 것.
- 모든 API 키는 GitHub Secrets / `.env`로 관리. 코드·저장소에 하드코딩 금지.
- 네이버가 메인, 구글 뉴스 RSS는 네이버 결과 부족 시에만 보충 호출. 구글 리다이렉트 링크 해제는 느리므로 최종 선택된 기사에 한해서만 처리.
- '24시간 신규' 필터: 각 소스의 발행일시(pubDate 등)로 지난 24시간만 통과.
- 중복 제거: 제목·URL 기준(같은 사건을 여러 매체가 보도).
- Currents / 각 무료 티어 한도는 가입 직전 재확인(조건 변동 가능).

## 13. 제작 순서 (단계별 — 이 순서로 만들 것)

- **1단계 (로컬 수동 실행)**: 키워드로 뉴스 수집 → Gemini로 요약·인사이트 생성 → JSON 저장까지. 여기서 **요약/인사이트 품질을 먼저 검증**하고 프롬프트를 다듬는다.
- **2단계 (웹사이트)**: 3개 탭 + summary 박스 + 버튼 + 펼치기 + 경제지표 카드/차트 구현. 1단계 JSON을 읽어 렌더.
- **3단계 (자동화·배포)**: GitHub Actions cron 연결 + 호스팅 배포 + Gmail SMTP 이메일 발송 추가.
- **4단계 (튜닝)**: 키워드 노이즈 제거, 인사이트 프롬프트 개선, 지표/레이아웃 조정.
